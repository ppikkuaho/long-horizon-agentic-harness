❯ Build a URL shortener.

⏺ Restated intent: Build a URL shortening service that maps long URLs to short codes and redirects visitors to the original destination.

  ---
  Clarifying Questions

  Before the decomposition is sound, I need answers to these load-bearing questions:

  1. Deployment & Scale

  - Traffic profile: personal/hobby, small team, or public-facing (millions of redirects/day)?
  - Deployment target: local-only, self-hosted server, or a cloud provider (Vercel, Fly, AWS, etc.)?

  2. Persistence

  - What database is acceptable? (SQLite for simple, Postgres/MySQL for multi-instance, Redis for high-speed redirects?)
  - Must data survive process restarts? (yes for any real use; confirming)

  3. Auth & Ownership

  - Anonymous use (anyone can shorten) or authenticated (login required)?
  - Should users be able to see/manage only their own links, or is it a shared admin pool?

  4. Custom Slugs

  - Auto-generated codes only (e.g. abc12), or allow user-defined vanity slugs (e.g. /my-blog)?

  5. Features Scope

  - Analytics: click counts? per-referrer/geo breakdown? or none?
  - Expiry: should links expire (by date or click count)?
  - API: REST API only, or also a web UI?

  6. Tech Preferences

  - Language/runtime preference (Node, Python, Go, other)?
  - Frontend: server-rendered HTML, or SPA (React/Vue)?
