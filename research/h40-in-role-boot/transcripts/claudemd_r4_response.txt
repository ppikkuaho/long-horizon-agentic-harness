⏺ Intent: Build a URL shortener service.

  ---
  Clarifying questions

  Before decomposition is sound, I need answers to these:

  1. Scale — personal/internal tool, or public-facing with potentially millions of slugs/redirects per day?
  2. Persistence — which datastore is in scope (Postgres, Redis, SQLite, DynamoDB, …)? Or is that an L3 decision?
  3. Auth — anonymous link creation, or authenticated users with link ownership/management?
  4. Custom slugs — must users be able to choose their own short code, or is auto-generation sufficient?
  5. Analytics — click counts only, or full event stream (referrer, geo, device)?
  6. Expiry — do links expire? By time, by click count, or never?
  7. Deployment target — what runtime/infra? (serverless, containerized, bare server, existing stack?)
  8. API surface — REST? GraphQL? CLI? Web UI? All of the above?

  First-cut area decomposition

  ┌───────────────────────┬───────────────────────────────────────────────────────────────────────────────┐
  │         Area          │                                Responsibility                                 │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Slug Generation       │ Produce unique, collision-free short codes; handle custom slugs               │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Link Store            │ Persist slug → long URL mappings; enforce TTL/expiry                          │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Redirect Engine       │ Handle inbound short-URL requests; 301/302 routing; performance-critical path │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Analytics Collector   │ Capture click events; decouple from redirect hot path                         │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Analytics Query       │ Aggregate and surface click data to callers                                   │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ API / Ingress         │ Public contract for create, read, delete, update operations                   │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ Auth & Ownership      │ (conditional on Q3) Identity, token issuance, link ownership scoping          │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
