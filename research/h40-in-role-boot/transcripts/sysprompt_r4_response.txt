вқҜ Build a URL shortener.

вҸә Intent

  Build a web service that accepts long URLs and returns short, redirectable aliases.

  ---
  Clarifying Questions

  Before decomposition is sound, I need answers to these load-bearing questions:

  Scale & Deployment

  1. Traffic target вҖ” personal/hobby (~100 req/day), small team (~10k/day), or public-facing (~millions/day)? This drives whether storage is SQLite-on-disk vs. a proper DB, and whether caching (Redis, etc.) is needed.
  2. Deployment target вҖ” local-only, self-hosted VPS, or a cloud platform (Fly, Railway, Vercel, AWS, etc.)?

  Persistence & Data

  3. Database вҖ” any preference (Postgres, SQLite, Redis, managed DB)? Or leave to L3 to choose?
  4. Link expiry вҖ” do short links expire (TTL), or do they live forever?
  5. Custom slugs вҖ” can users choose their own alias (e.g. /my-link), or are all slugs auto-generated?

  Auth & Access Control

  6. Authentication вҖ” is this open (anyone can create links), single-user (just you), or multi-tenant (accounts with per-user link ownership)?
  7. Admin/management вҖ” does there need to be a UI or API to list, edit, or delete links?

  Analytics

  8. Click tracking вҖ” should the system record click counts, timestamps, referrers, or geo? Or pure redirect, no analytics?

  Interface

  9. API vs. UI вҖ” REST/JSON API only, a web UI (form to shorten + dashboard), or both?
  10. Stack constraint вҖ” any language/framework preference, or is L3 free to propose?
