❯ Build a URL shortener.

⏺ Intent

  Build a URL shortener service that maps short codes to long URLs and redirects users on access.

  ---
  Clarifying Questions

  Before decomposition is sound, I need answers on these load-bearing unknowns:

  Scale & Deployment

  1. Traffic profile — personal/hobby tool, small team, or public-facing service? (drives caching, infra choices)
  2. Deployment target — local process, single server, containerized, serverless, managed cloud (Vercel, Fly, etc.)?
  3. Expected volume — reads/writes per day? Any burst scenarios?

  Persistence

  4. Database preference — bring-your-own (Postgres, MySQL, Redis, SQLite) or let the design choose?
  5. Durability requirement — is data loss on restart acceptable (rules out in-memory-only)?

  Auth & Access Control

  6. Who can create short links? — anonymous public, authenticated users only, or API-key gated?
  7. Is there a user/account concept — i.e., do owners manage their own links?

  Feature Scope

  8. Custom slugs — can creators supply a vanity code (/my-brand) or is it always auto-generated?
  9. Link expiry — do links expire (TTL, click-count, or date)?
  10. Analytics — click counts only, full referrer/geo/UA breakdown, or none?
  11. Admin/management UI — web dashboard, CLI, REST API only, or some combination?
  12. QR code generation — in or out of scope?

  Non-Functionals

  13. Short code length & alphabet — any constraints (e.g., no ambiguous chars like 0/O)?
  14. Collision strategy — retry on hash collision, pre-generated pool, sequential?
