# Tame ↔ Wicked Problem
**Family:** Environmental / Context Dimensions
**Poles:** Tame ↔ Wicked

## What This Is
Tame problems have clear formulations, definite stopping criteria, and right answers that can be verified — you know when you've solved them. Wicked problems resist even being defined, change as you work on them, have no stopping rule, and have no objectively correct solution — every attempted solution changes the problem itself.

Rittel and Webber identified wickedness in planning problems, but it generalizes: poverty, organizational culture, education reform, and most important social challenges are wicked. The danger is applying tame-problem methods to wicked problems — expecting clear formulation, definitive solutions, and verifiable answers when none of these are available. Wicked problems require iterative, adaptive approaches that accept ongoing engagement rather than resolution.

## Signs You're At Each Pole
**Tame:** The problem can be clearly stated. You'll know when it's solved. There's a right answer. Similar problems have been solved before with known methods.
**Wicked:** The problem resists definition — different stakeholders see different problems. Every solution attempt changes the problem. There's no clear stopping point. Previous solutions haven't worked because the problem shifts.

## Steering Moves
- Before committing to an approach, ask: "Will I know when this is solved?" If yes, treat it as tame and drive toward the known solution; if no, switch to probe-and-adapt mode — design the smallest intervention that would teach you something, run it, then reassess.
- When a wicked-problem intervention produces unexpected results, resist redesigning the whole plan; instead, name what the result revealed about the problem's structure, update your problem definition to include that, and design your next small probe from the updated definition.
- When stakeholders describe what looks like the same situation as fundamentally different problems, map each version explicitly — write down each party's problem statement side by side — and build your intervention to address the overlap; if there is no overlap, you've confirmed the problem is wicked enough that a single solution won't work, so pursue parallel small probes targeting each framing separately.
