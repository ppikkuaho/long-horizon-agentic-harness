## THE BUILD LOOP (how every increment is done — carry this exactly)
1. **Tester** (workflow) writes the FROZEN pytest acceptance from the increment's Done-test — RED first.
2. **Builder** (separate) implements to GREEN; cannot edit the frozen tests; escalates a wrong test.
3. **Reviews** (parallel): architecture cleanliness + load-bearing coverage ("is *tests-pass == correct*?").
4. **MY verification gate** (the orchestrator, NOT delegated): re-run green, then **mutation-test** — inject
   each plausible bug, confirm a test goes red; a SURVIVING mutant means the tests are theater → I strengthen
   them. Fix any review-flagged real issue. ONLY commit when the increment's condition genuinely holds.
5. Commit on `main`; messages end `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Workflows can't be trusted blind** — I grep/read ground truth myself (Lesson 4); a subagent's
  "I verified / tests pass" is confabulation-prone. The mutation gate is the teeth behind "tests==correct".
